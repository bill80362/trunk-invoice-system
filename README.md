# 貨運請款單管理系統

基於 Laravel 13 + Filament v5 建構的貨運請款單管理系統，提供基礎資料維護、費率管理、請款單建立與列印等功能。

## 系統需求

- PHP 8.4+
- Composer
- Node.js & NPM
- SQLite（預設）或其他 Laravel 支援的資料庫

## 安裝

```bash
# 安裝 PHP 依賴
composer install

# 複製環境設定
cp .env.example .env
php artisan key:generate

# 建立資料庫並填充測試資料
php artisan migrate --seed

# 安裝前端依賴（如需自訂樣式）
npm install && npm run build
```

## 啟動

```bash
php artisan serve
```

開啟瀏覽器前往 http://localhost:8000/admin

## 功能說明

### 基礎資料管理（側欄「基礎資料」）

- **司機**：維護司機姓名與電話
- **托運方式**：維護貨運方式類別（如：整車、零擔）
- **地點**：維護起點與目的地名稱
- **貨主**：維護客戶名稱、聯絡人、電話、地址

> 已被請款單引用的基礎資料無法刪除。

### 費率表（側欄「費率設定」）

設定「起點 → 目的地 + 托運方式」對應的基礎運費。同一組合不可重複建立。

### 系統設定（側欄「系統設定」）

- **附加站點費用**：行程有多個目的地時，每個額外站點加收的費用
- **請款方預設值**：公司名稱、地址、電話（建立請款單時自動帶入）

### 請款單管理（側欄「請款管理」）

#### 建立請款單

1. 選擇**貨主**、填寫**年份**與**月份**（同一貨主同年月僅能一張）
2. 請款方資訊會自動帶入系統設定值，可手動修改
3. 儲存後進入編輯頁面，即可新增行程明細

#### 行程明細

每筆行程包含：
- **日期**：貨運日期
- **起點**：出發地
- **托運方式**：運送方式
- **司機**：負責司機
- **目的地**：可新增多個目的地（嵌套列表）
- **重量**：貨物重量（選填）
- **運費**：根據費率表自動計算，也可手動覆寫

**運費自動計算規則**：
- 查詢「起點 → 第一個目的地 + 托運方式」的基礎運費
- 若有額外目的地，加上「額外站點數 × 附加站點費用」

行程支援拖曳排序。

#### 確認與解除鎖定

- 點擊 **「確認請款單」** → 請款單鎖定為唯讀，記錄確認時間
- 點擊 **「解除鎖定」** → 恢復為草稿，可繼續編輯

#### 列印

點擊 **「列印」** 按鈕會在新分頁開啟列印預覽頁面，使用瀏覽器列印功能即可輸出。草稿狀態會顯示「草稿」浮水印。
